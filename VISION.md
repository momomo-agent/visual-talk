# Visual Talk — Vision

## 一句话

从 AI 对话工具到 AI-native OS。

## 核心信念

UI 是 AI 的第一语言。不是聊天框里打字，不是 Markdown 渲染——是 AI 直接用空间构图表达思想。

## 三个阶段

### Phase 1: 对话工具（现在）
- 你说话 → AI 用 UI 回应 + 语音伴随
- Timeline 分支树管理对话历史
- Canvas 空间构图，不是组件列表
- 文本协议驱动，streaming 友好

### Phase 2: 环境智能
- **多模态输入** — 图片、截图、手绘、摄像头
- **持久化** — 跨 session 记忆，对话树持久存储
- **主动感知** — AI 观察用户行为（注意力、犹豫、模式）
- **主动推送** — AI 可以不被触发就往画布上放东西

### Phase 3: AI-native OS
- **事件循环** — 时间事件、注意力事件、外部事件驱动
- **能力即插件** — Skill 声明式加载，不是 App 安装
- **注意力管理** — Canvas z-depth 就是优先级系统
- **进程 = 意图** — Timeline node 是意图的进程，不是文件的进程

## 架构映射

| 传统 OS | Visual Talk |
|---------|-------------|
| 文件系统 | Timeline（按对话树组织信息） |
| 窗口管理器 | Canvas（空间构图 + 注意力管理） |
| 命令行 | 语音/文字输入（意图，不是指令） |
| App | Skill（prompt 里的能力声明） |
| 进程 | Timeline node（意图的执行单元） |
| 系统调用 | 文本协议 `<!--vt:card-->` |

## 设计原则

1. **表达 > 执行** — AI 在用 UI "说话"，不是在"调用组件"
2. **空间 > 列表** — 卡片之间有位置、层次、时间关系
3. **流动 > 原子** — streaming 构图，说到哪出到哪
4. **主动 > 被动** — AI 知道什么时候该说话、该闭嘴、该帮忙
5. **意图 > 操作** — 用户表达想要什么，不是怎么做

## 技术栈

- 纯前端（Vue 3 + Pinia + Vite）
- LLM: Anthropic 协议
- TTS/STT: OpenAI 兼容
- 搜索: Tavily
- 部署: GitHub Pages
